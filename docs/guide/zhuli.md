# 助播设置

助播设置用于配置助播的自动接话功能。当主播讲完一段话后，系统可以自动播放助播的接话语音。

## 功能说明

### 什么是助播接话？

主播讲完一段话后，系统自动播放预设的助播语音，形成"主播-助播"的对话效果，让直播更生动。

### 为什么要用助播接话？

- **提升互动** - 形成对话效果，更吸引观众
- **稳定节奏** - 避免冷场
- **节省时间** - 无需手动切换
- **专业感** - 看起来像有真实助播

---

## 配置步骤

### 第1步：准备助播语音

1. 录制或准备助播的语音文件
2. 转换为MP3或WAV格式
3. 放入 `zhuli_audio/` 目录

### 第2步：进入助播设置

1. 点击左侧菜单的 **助播设置**
2. 查看当前配置

### 第3步：配置接话规则

1. 设置接话触发条件
2. 选择接话语音
3. 设置接话延迟（可选）

### 第4步：保存并测试

1. 点击 **保存** 按钮
2. 在 **AI工作台** 点击 **试听** 测试
3. 确认效果满意

---

## 助播语音库

### 目录结构

```
zhuli_audio/
├── 回复主播
│   ├── 测试语音回答_助理1.wav
│   ├── 测试语音回答_助理2.wav
│   └── 测试语音回答_助理3.wav
└── 其他分类
    └── 语音文件.mp3
```

### 语音文件要求

- **格式** - MP3、WAV优先
- **时长** - 建议5-30秒
- **音量** - 与主播语音保持一致
- **质量** - 清晰无杂音

---

## 接话规则

### 触发条件

| 条件 | 说明 |
|------|------|
| 主播讲完 | 主播停止讲话后触发 |
| 延迟时间 | 可设置延迟N秒后播放 |
| 随机选择 | 从语音库中随机选择 |

### 配置示例

**场景**：产品讲解

```
触发条件：主播讲完后2秒
接话语音：从"回复主播"文件夹随机选择
效果：形成"主播讲解 → 2秒停顿 → 助播接话"的效果
```

## 常见问题

### Q: 如何录制助播语音？

A: 
1. 使用手机或电脑录音软件
2. 录制清晰的语音
3. 转换为MP3或WAV格式
4. 放入 `zhuli_audio/` 目录

### Q: 可以使用TTS生成的语音吗？

A: 可以。系统支持任何格式的音频文件。

### Q: 如何修改接话延迟？

A: 
1. 进入 **助播设置**
2. 修改延迟时间
3. 点击 **保存**

### Q: 如何关闭助播接话？

A: 
1. 进入 **AI工作台**
2. 关闭 **助播接话** 开关

### Q: 可以自定义接话规则吗？

A: 可以。在 **助播设置** 中配置触发条件和语音选择。

## 最佳实践

### 1. 准备多条接话语音

- 准备3-5条不同的接话语音
- 系统会随机选择
- 避免重复感

### 2. 保持语音风格一致

- 助播语音应与主播风格搭配
- 音量、语速、语调要协调
- 形成统一的直播风格

### 3. 设置合理的延迟

- 延迟太短：显得生硬
- 延迟太长：显得冷场
- 建议：1-3秒

### 4. 定期更新语音库

- 定期添加新的接话语音
- 删除过时的语音
- 保持内容新鲜

## 故障排查

### 助播接话没有触发

**可能原因**：
1. 助播接话开关未打开
2. 语音文件不存在
3. 触发条件配置错误

**解决方案**：
1. 进入 **AI工作台** 确认开关已打开
2. 检查 `zhuli_audio/` 目录中是否有语音文件
3. 检查触发条件配置

### 接话语音没有声音

**可能原因**：
1. 系统音量关闭
2. 语音文件损坏
3. 文件格式不支持

**解决方案**：
1. 检查系统音量
2. 尝试用其他播放器打开文件
3. 转换为MP3或WAV格式
